Resumen Ejecutivo

Viveprop Evaluaciones Comerciales es una aplicación web de tasación y caso de negocio inmobiliario orientada al mercado chileno. Permite a corredores y tasadores valorizar propiedades (casas y departamentos) mediante metodologías formales reconocidas en Chile, generar un análisis de rentabilidad por arriendo y descargar un informe profesional en PDF.

✓ Principios de diseño 100 % client-side · Sin backend · Sin autenticación · Sin persistencia de datos entre sesiones · Todos los cálculos ocurren en el navegador del usuario

Funcionalidades principales

  • Wizard guiado de 7 pasos con indicador de progreso
  • Soporte dual: Casa (REP + Estadístico + Media) y Departamento (Competencia Directa)
  • Carga automática del valor UF del día desde mindicador.cl
  • Tabla Ross-Heidecke completa embebida para calcular depreciación de construcciones
  • Análisis de arriendos con cálculo de cap rate bajo tres escenarios de vacancia
  • Vista previa del informe en pantalla
  • Descarga del informe completo en PDF (formato carta, con paginación automática)

Métodos de valorización implementados

TipoMétodoNorma de referenciaOutput
Casa REP (Reposición) Valorización por costo de reposición física Valor terreno + construcción depreciada
Casa Estadístico (STDEV) Mediana ± desviación estándar de comparables Intervalo estadístico de valor terreno
Casa MEDIA Promedio ponderado REP + STDEV Valor final sugerido
Departamento Competencia Directa (COMP) Comparación de mercado con ajuste por factores UF/m² ajustado → valor total

2.2 Arquitectura

La aplicación es una Single Page Application (SPA) con una única ruta (/). No existe backend; toda la lógica corre en el navegador. El estado del wizard vive en el componente raíz Index.tsx y se pasa hacia abajo via props.

Patrón de estado Estado local con useState en Index.tsx. No se usa Redux ni Context. Al hacer "Nueva Tasación" el estado se resetea completamente a valores por defecto. No hay persistencia (localStorage ni sessionStorage).

Flujo de datos

Index.tsx (estado global)
  ├── StepIndicator (UI de navegación, recibe step + propertyType + onReset)
  ├── PropertyTypeSelector (Paso 0, emite onSelect)
  ├── ConfigStep (Paso 1, recibe/emite params)
  ├── SubjectStep (Paso 2, recibe/emite subject)
  ├── ComparablesCasaStep | ComparablesDeptStep (Paso 3, recibe/emite comps)
  ├── ResultsStep (Paso 4, recibe result calculado)
  ├── RentalStep (Paso 5, recibe/emite arriendoAnalysis)
  └── ReportStep (Paso 6, consume todo el estado para mostrar y exportar)

lib/engines.ts  ← funciones puras de cálculo (sin estado)
lib/types.ts    ← interfaces TypeScript + constantes + helpers
lib/pdfExport.ts← genera PDF usando jsPDF

Especificación Funcional

La app implementa un wizard lineal de 7 pasos. El usuario no puede saltar pasos, pero puede retroceder. El estado global se preserva al navegar hacia atrás.

Paso 0

Tipo de Propiedad

Pantalla de inicio. Selección exclusiva: Casa o Departamento.

Paso 1

Parámetros de Mercado

Valor UF, factores de negociación, costos unitarios.

Paso 2

Datos del Inmueble

Identificación, superficies, construcción (Casa) o factores (Depto).

Paso 3

Comparables

Tabla editable de propiedades comparables con columnas calculadas en tiempo real.

Paso 4

Valores de tasación por los tres métodos (Casa) o Competencia Directa (Depto).

3.4 Paso 3 — Ingreso de comparables

Tabla de comparables Casa (máx. 12)

Columnas de entrada (editables): Activo, Precio Pub. (UF), m² Terreno, m² Construidos, Año, Material, Estado, Costo UF/m², Estacionamientos, Bodegas.
Columnas calculadas en tiempo real (readonly): Precio Venta UF, % Vida, FK RH, Terreno UF/m² (columna dorada — input clave para REP).

⚠ Alerta de fila Si un comparable está activo y tiene precio, pero su resultado de Terreno UF/m² es 0 o negativo (precio neto insuficiente), la fila se resalta en rojo y la celda muestra "—". Esa fila no contribuye al cálculo.

Tabla de comparables Departamento (máx. 6)

Columnas de entrada: Activo, Precio (UF), m² Útiles, m² Terraza, F.Estado, F.Orient., F.AntPub., Estacionamientos, Bodegas.
Columnas calculadas: Precio Venta UF, m² Total, Precio Neto UF, UF/m² Nominal, UF/m² Ajustado (columna dorada — input clave para COMP).

Iniciadores por defecto: Casa = 4 filas; Depto = 3 filas. Se pueden agregar y eliminar filas (excepto bajar del mínimo inicial).

3.5 Paso 4 — Resultados de tasación

Se muestra el resultado del motor de cálculo. El cálculo se ejecuta al presionar "Calcular" al final del paso 3.

Casa — 3 métodos

  • Card destacada con MEDIA MIN / ⭐ PROM / MAX en UF (y CLP si aplica)
  • Tabla detalle: 3 componentes (Terreno, Construcción, Adicionales) × 6 columnas (REP MAX/MIN, STDEV MAX/MIN, MEDIA MAX/MIN)
  • 3 tarjetas métricas: Valor Tierra REP (MAX/MIN), Intervalo STDEV (MAX/MIN), Mediana ± Desv.

Departamento — Competencia Directa

  • Card con COMP MIN / ⭐ PROM / MAX
  • Tabla: 4 componentes (Útil, Terraza, Estacionamientos, Bodegas) × 3 columnas (MAX, MIN, PROM)
  • 2 tarjetas: CD Últimos 3 (UF/m²), Promedio General (UF/m²)

3.6 Paso 5 — Análisis de arriendos y rentabilidad

Permite ingresar comparables de arriendo y calcular el cap rate del inmueble tasado.

  • Tabla de arriendos: referencia (texto libre), m² Totales, arriendo publicado, moneda (CLP o UF). Mínimo 3 comparables.
  • Si algún comparable está en UF y no se ingresó el valor UF del día: alerta amarilla
  • Si hay valor UF: columna adicional mostrando el arriendo en CLP
  • Input de valor de propiedad en UF (para el cap rate)
  • Selector de vacancia: 1 / 2 / 3 meses sin arriendo por año
  • Vista previa en tiempo real (aparece al tener 3+ comparables válidos): métricas de arriendo promedio, valor propiedad, gastos anuales; tabla de los 3 escenarios de vacancia.

3.7 Paso 6 — Informe y descarga PDF

Muestra el informe completo en pantalla como preview y permite descargarlo en PDF.

Secciones del informe:

  1. Encabezado navy: título, tipo de propiedad, fecha, UF del día, datos del propietario
  2. I. Análisis de la Propiedad: todos los datos del inmueble sujeto
  3. II. Factibilidad Comercial: expectativa vendedor, valor tasación, avalúo fiscal, factores diferenciadores
  4. Arriendos comparativos (si se calcularon)
  5. III. Estrategia de Precio: MIN / ⭐ PROM / MAX en UF y CLP
  6. Impacto de la Vacancia: tabla de 3 escenarios con cap rate
  7. IV. Conclusiones: párrafo autogenerado con los valores clave

Botones: ← Volver | Descargar PDF | Nueva Tasación (resetea wizard).

5.1 Tabla Ross-Heidecke (FK_RH)

La tabla Ross-Heidecke es el estándar chileno de depreciación de construcciones. Se indexa por % de vida útil consumida (eje X, pares de 2 a 100) y estado de conservación (eje Y, A–H). El valor de la tabla es K (% de depreciación).

FK_RH = (100 − K) / 100
donde K = RH_DATA[pct_vida_redondeada][índice_estado]

% vida útil = (año_actual − año_construcción) / vida_útil_material × 100
pct_vida_redondeada = redondear al par más cercano, mín 2, máx 100

5.2 Procesamiento de comparable Casa

Cada fila de comparable se transforma mediante las siguientes fórmulas:

edad = año_actual − comp_year
% vida útil = (edad / vida_util_material) × 100
FK_RH = tabla_RH(% vida, estado) // (100 − K) / 100

precio_venta_uf = comp_price_pub_uf × negotiation_factor

costo_dep_uf_m2 = comp_cost_uf_m2 × FK_RH
costo_total_uf = comp_built_m2 × costo_dep_uf_m2

precio_neto_uf = precio_venta_uf − comp_parking × parking_uf_casa − comp_storage × storage_uf

terreno_uf_m2 = (precio_neto_uf − costo_total_uf) / comp_land_m2 // usado en REP

precio_stdev_uf = comp_price_pub_uf × (FK_RH − stdev_factor_adj)
precio_neto_stdev = precio_stdev_uf − comp_parking × parking_uf_casa
terreno_stdev_m2 = precio_neto_stdev / comp_land_m2 // usado en STDEV

dias_mercado = (hoy − comp_pub_date) en días // opcional, informativo
Nota Si el resultado de terreno_uf_m2 o terreno_stdev_m2 es negativo (precio neto insuficiente para cubrir el costo de construcción), se registra como 0 y la fila no contribuye al cálculo de REP/STDEV. La UI lo muestra en rojo.

5.3 Método REP (Reposición Física)

Valora el inmueble como suma del valor del terreno (extraído de comparables) más el costo de construcción a nuevo depreciado.

// Valor del terreno
activos_REP = comparables con terreno_uf_m2 > 0 (últimos 4)
vt_MAX = promedio(terreno_uf_m2 de activos_REP)
vt_MIN = vt_MAX × 0.95
vt_PROM = (vt_MAX + vt_MIN) / 2

// Superficies del sujeto
m²_c = m²_utiles + m²_terraza × peso_terraza_area + m²_loggia
m²_t = m²_terreno
m²_ad1 = adicional_1_m2 | m²_ad2 = adicional_2_m2

// Escenario MAX
terreno_uf = m²_t × vt_MAX
construccion_uf = m²_c × costo_REP × 1.00
adicional_uf = m²_ad1 × costo_adicional + m²_ad2 × costo_adicional
REP_MAX.total_uf = terreno_uf + construccion_uf + adicional_uf

// Escenario MIN: usar vt_MIN y factor 0.95 en construcción
REP_MIN.total_uf = m²_t × vt_MIN + m²_c × costo_REP × 0.95 + adicionales

REP_PROM.total_uf = (REP_MAX.total_uf + REP_MIN.total_uf) / 2

5.4 Método Estadístico (STDEV)

Aplica estadística descriptiva sobre los valores de terreno para determinar un intervalo de confianza.

activos_STDEV = comparables con terreno_stdev_m2 > 0 (últimos 3) — requiere ≥ 2

mediana = mediana de terreno_stdev_m2 de activos_STDEV
desv = desviación estándar muestral de terreno_stdev_m2

intervalo_MAX = mediana + desv
intervalo_MIN = mediana − desv

// Escenario MAX (igual lógica que REP pero usando costo_STDEV e intervalo_MAX)
STDEV_MAX.total_uf = m²_t × intervalo_MAX + m²_c × costo_STDEV × 1.00 + adicionales
STDEV_MIN.total_uf = m²_t × intervalo_MIN + m²_c × costo_STDEV × 0.95 + adicionales
STDEV_PROM.total_uf = (MAX + MIN) / 2
Error Si hay menos de 2 comparables activos con terreno_stdev_m2 > 0, el sistema lanza error: "Se requieren al menos 2 comparables activos para el Método Estadístico."

5.5 Método MEDIA (promedio REP + STDEV)

Combina ambos métodos promediando valores de terreno y costos de construcción.

// Escenario MAX
vt = (vt_MAX_REP + intervalo_MAX_STDEV) / 2
cc = (costo_REP + costo_STDEV) / 2
terreno_uf = m²_t × vt
construccion_uf = m²_c × cc × 1.00
MEDIA_MAX.total_uf = terreno_uf + construccion_uf + adicionales

// Escenario MIN: usar vt_MIN_REP + intervalo_MIN_STDEV, factor 0.95
MEDIA_MIN.total_uf = m²_t × vt_MIN + m²_c × cc × 0.95 + adicionales

MEDIA_PROM.total_uf = (MEDIA_MAX + MEDIA_MIN) / 2

El Método MEDIA es el valor sugerido final para casas. Se presenta como el resultado principal en el informe.

6.2 Procesamiento de comparable Departamento

precio_venta_uf = comp_price_pub_uf × (comp_factor_estado − 0.05)
m2_total = comp_util_m2 + comp_terrace_m2 × terrace_weight_depto_area
precio_neto_uf = precio_venta_uf − comp_parking × parking_uf_depto − comp_storage × storage_uf

valor_nominal_uf_m2 = precio_neto_uf / m2_total
valor_ajustado_uf_m2 = valor_nominal_uf_m2 × comp_factor_estado × comp_factor_orient × comp_factor_antipub

6.3 Método Competencia Directa (COMP)

Requiere mínimo 3 comparables activos con valor_ajustado_uf_m2 > 0.

uf_m2_promedio = promedio(valor_ajustado_uf_m2 de todos los activos)
cd_uf_m2 = promedio(valor_ajustado_uf_m2 de los últimos 3 activos) // base del cálculo

// Factores del sujeto: fe = factor_estado, fo = factor_orient, ff = factor_franja

// Escenario MAX (orientación como techo)
uf_m2_util_MAX = cd_uf_m2 × fe × fo
uf_m2_terraza_MAX = uf_m2_util_MAX × terrace_price_weight_depto
util_uf_MAX = m²_utiles × uf_m2_util_MAX
terraza_uf_MAX = m²_terraza × uf_m2_terraza_MAX
est_uf_MAX = estacionamientos × parking_uf_depto
bod_uf_MAX = bodegas × storage_uf
COMP_MAX.total_uf = util_uf + terraza_uf + est_uf + bod_uf

// Escenario MIN (franja como suelo)
uf_m2_util_MIN = cd_uf_m2 × fe × ff
est_uf_MIN = estacionamientos × parking_uf_depto × ff
bod_uf_MIN = bodegas × storage_uf × ff
COMP_MIN.total_uf = util_uf_MIN + terraza_uf_MIN + est_uf_MIN + bod_uf_MIN

COMP_PROM.total_uf = (COMP_MAX + COMP_MIN) / 2
Error Si hay menos de 3 comparables activos con valor_ajustado_uf_m2 > 0, el sistema lanza: "Se requieren al menos 3 comparables activos para la Competencia Directa."

Especificación Lógica — Análisis de arriendos

7.1 Homologación de monedas

Cada comparable de arriendo puede estar en CLP o UF. Si está en UF y existe el valor UF del día, se convierte: arriendo_clp = arriendo_UF × uf_value_clp. Si está en UF y no hay valor UF, el cálculo arroja error.

7.2 Cálculo de rentabilidad y cap rate

// Requiere ≥ 3 comparables con arriendo_clp > 0
arriendo_promedio_clp = promedio(arriendo_clp de comparables válidos)
arriendo_promedio_uf = arriendo_promedio_clp / uf_value_clp (si hay UF)
valor_propiedad_clp = valor_propiedad_uf × uf_value_clp (si hay UF)

gastos_anuales_clp = arriendo_promedio_clp × 12 × 0.10 // 10% de gastos anuales

// Para cada escenario de vacancia (meses = 1, 2 o 3):
meses_ocupados = 12 − meses
ingresos_anuales_clp = arriendo_promedio_clp × meses_ocupados
ingreso_neto_anual_clp = ingresos_anuales_clp − gastos_anuales_clp

cap_rate = (arriendo_promedio_clp / valor_propiedad_clp) × meses_ocupados × 0.90 × 100
// Factor 0.90 representa el 90% de ocupación efectiva ponderada
